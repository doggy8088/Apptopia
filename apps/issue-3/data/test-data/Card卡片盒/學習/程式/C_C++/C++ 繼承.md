# 繼承

基本上繼承跟C#一樣，不同在於
- C++容許多重繼承
- C++繼承可以分權限的

## C++權限繼承
什麼叫權限繼承(名字自己掰的)，

- C#繼承就是父類除private所有東西都會移到子類
```C
Class B : A =>這是C#繼承方式
```

- C++則是
```C
Class B : public A
Class B : protected A 
Class B : private A 
```
會在繼承前面多加上關鍵字
以private舉例，B繼承了A的所有東西，但是A裡面所有東西(包含變數、函式)通通變成private
public 則是維持原樣(跟C#一樣)
protected則是將A裡面public換成protected這樣
如果繼承前面未加上關鍵字，預設是private


## 多重繼承
多重繼承是C++的一個特性，是早期時候留下來的
使用方式就一直接續在後面
```C
Class Z : public A , B, C private D
```

多重繼承雖然在製作上方便
但在[[編寫程式的邏輯：如何用物件導向實作複雜的業務需求|編寫程式的邏輯]]書中提到
會形成結構僵化(繼承的問題)
以及最麻煩的死亡三角，以下圖來看
A function 分別繼承出去，但最下面的函式多重繼承，並使用A function
但這時，該使用誰的 A function
![](../../../../Source來源筆記/圖片/C++%20繼承/繼承死亡三角.png)

隨時代進步，程式語言都取消多重繼承，但也是有繼續使用的
例如: Dart語言就是這樣，至於如何解決死亡三角，就是如果有衝突函式一律呼叫繼承順序最後一個
